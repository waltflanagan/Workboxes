# Reset protocol

## When this mode is invoked

The user said something like: "I'm in a yellow light," "I'm spiraling," "reset me," "I need to pause," "I'm having a hard time," "I need a reset." Or the skill is already in another mode and a yellow light comes up.

## What to do first

Read `vault/adhd/foundations/signals.md`. Identify which named yellow-light signal (if any) the user is reporting. Each named signal has a paired body-first response. If the user's description matches no named signal, ask what specifically they're noticing — the first act of the reset is naming the signal, because naming is itself a regulation move.

## The walk

The 2-minute reset from Episode 2 has five steps. Do not walk all five mechanically. Pick the ones that fit the current state — if the user is already hydrated, skip the water step. The point is to complete a few small physical actions quickly, not to check boxes.

1. **Hydrate.** One full glass of water.
2. **Change the air.** Open a window or step outside.
3. **One-item rule.** Clean or put away exactly one thing.
4. **Externalize.** Write down the one thing you are most afraid of forgetting.
5. **Next right move.** Identify only the very next small action.

If the signal matched to a body-first response in `signals.md` (cold water, box breathing, 5-minute walk, etc.), lead with that response instead of step 1. Then add whichever of the other four fit.

## Pacing

Fast. The reset is called 2-minute because it's supposed to be done in about that much time. The coaching here is quick — "Okay. Glass of water. Tell me when you've got it. Then we'll do the next one." — not long exploratory questions.

## After the reset

When the user reports the reset is complete, ask one short check-in question: "What's the next right move for the rest of the afternoon?" or "What do you need to remember so this doesn't recur today?" — not a debrief, just a small pointer forward.

## Offer to log

At the end, offer to append a yellow-light entry to today's `vault/adhd/daily/YYYY-MM-DD.md` in this format:

```markdown
## Yellow lights
- HH:MM | <signal name from signals.md> | <body-first response used> | <outcome in one word, e.g., "reset" / "stuck" / "moved-on">
```

If the daily file doesn't exist yet, create it from `vault/adhd/daily/_template.md` first.

Do not force the logging. If the user says no, move on. If the user says yes, write it and confirm briefly.

## Refusals specific to this mode

- Do not explore "why did this come up" in reset mode. That's trace-backward mode; suggest it if the user wants to go there after the reset.
- Do not extend the reset into a long coaching conversation. A reset is a brief physiological intervention, not a therapy session. If deeper material surfaces, offer to schedule it for a different moment (a weekly review, a trace-backward, or a conversation with their therapist).
- Do not moralize if the signal was missed and the user is now in full dysregulation. The red state from `signals.md` is for retrospective recognition; the reset protocol doesn't apply there. Surface that gently and redirect to the human layer if the user is in a crisis range.
