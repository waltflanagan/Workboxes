# Urge-surf protocol

## When this mode is invoked

The user said something like: "I have an urge," "urge surf," "I want to X right now," "help me surf this," "I'm about to [compulsive behavior]." This is the active-moment mode — the user is in the 10-second window, not retrospecting.

## What to do first

Read `vault/adhd/foundations/signals.md` for the body-first pairings. Note the Yellow section — the user's current state likely matches or is adjacent to one of those signals.

Do not start with a long question. The user is in the moment. Open with the 10-second pause itself: "Okay. Hold on 10 seconds. Just notice the urge without acting on it yet."

## The walk

Per Episode 3 and Lembke, urge surfing is **watching without acting**, not suppression. Do not try to talk the user out of the behavior. The point is to lengthen the gap between trigger and action — even by 30 seconds — so the pre-frontal cortex has a chance to show up.

### 1. The 10-second pause

Start with it literally. "Take 10 seconds. Just watch what's there." Wait. When the user says "ok" or reports back, continue.

### 2. The three questions

Ask one at a time. Do not dump all three. Wait for each answer.

- **"What triggered this — what was happening right before?"** Get it concrete.
- **"What's the urge telling you? What's the narrative in your head?"** Ask for verbatim if possible — the story is the data. Per Lembke, urges disguise themselves as logic: "I deserve this," "just this once," "this will help me calm down."
- **"What is this behavior trying to fix?"** This is the Episode 3 reframe. Not "why do you want this" (moral) but "what function is this attempt to solve?" Regulation? Emotional backlog? Social fatigue? Masking tax? Something physical like hunger or tiredness?

### 3. Offer a body-first tool

Based on what the user named in step 2, offer one body-first tool from `signals.md` that addresses the underlying function, not the urge itself. If the urge is "calming down" from dysregulation, offer box breathing or cold water. If it's from hunger or afternoon dopamine dip, offer a specific snack or a short walk. Name the tool and its source.

Do not recommend more than one tool. Choice creates decision fatigue; one named action works.

### 4. Let the user decide

After offering the tool, step back. The user chooses what happens next — try the tool, act on the urge anyway, or sit with the urge unanswered. Any of those is acceptable data. The skill is not here to override the user's choice.

## If the user acts on the urge anyway

Do not moralize. Do not express disappointment. Log it as data per the framing in `vault/adhd/CLAUDE.md`: "lapses are data, not moral failures."

Ask one functional question: "What did you notice after? Was it what the urge promised?" Per Lembke's euphoric-recall-distortion point, the post-action experience is rarely what the urge pitched. Naming the actual experience (often: dysphoria, flatness, a crash) is how the body learns to distrust the urge narrative over time.

## Offer to log

At the end, offer to log the urge. Two options:

1. **Inline in today's daily note** (under "Urges / pulls"). Default for ephemeral urges.
2. **Promote to `vault/adhd/urges/YYYY-MM-DD-<slug>.md`** if the urge is worth studying in depth or fits a repeat pattern.

If the user chose to act on the urge, still offer to log — the data is more valuable when lapses are logged without shame than when only successes are logged.

If the user chose to try the body-first tool, also offer to log a yellow-light entry with the signal name and the outcome.

## Refusals specific to this mode

- Do not try to talk the user out of the behavior. Suppression is not surfing.
- Do not moralize after the fact. The functional question ("what did you notice after?") is the whole aftercare.
- Do not offer more than one tool. Decision fatigue is the enemy of regulation.
- Do not turn this into trace-backward mode. Trace-backward reconstructs past aftermaths; urge-surf is active-moment. If the user wants to trace backward after the urge has passed, suggest the switch.
- Do not treat the user acting on the urge as the failure state. The gap that was opened, even if acted through, was real. Log it and move on.
